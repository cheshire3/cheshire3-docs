Cheshire3 Objects: Parser
Description

Normally a simple wrapper around an XML parser, these objects can be viewed as Record Factories. They take a Document containing some XML and produce the equivalent Record.

Base Implementations

Module: cheshire3.parser

  • MinidomParser
    Python standard library's Minidom parser - creates MinidomRecorda
  • SaxParser
    Default SAX based parser - creates SaxRecords
  • StoredSaxParser
    Parser to create SAX records from a serialization in a recordStore - creates SaxRecords
  • LxmlParser
    Libxml2 DOM parser - creates LxmlRecords
  • LxmlHtmlParser
    Libxml2 DOM parser for HTML (i.e. tolerates imperfectly formed XML e.g. unclosed <br>) - creates LxmlRecords
  • PassThroughParser
    Parser to simply copy the data from a document (e.g. list of sax events or a DOM tree) into an appropriate record object.
  • MarcParser
    Parser for MARC data - creates MarcRecords
Sub-package Implementations

Sub-package: graph
Module: cheshire3.graph.parser

  • RdfGraphParser
    RDF serialization parser - creates GraphRecords.
    Parser must be configured with appropriate format setting, currently supports :
    • xml
    • trix
    • nt
    • n3
    • rdfa
  • RdfaParser
    RDFa serialization parser that uses pyRdfa - creates GraphRecords.
