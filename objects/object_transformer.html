Cheshire3 Objects: Transformer
Description

A Transformer is the opposite of a Parser. It takes a Record and produces a Document. In many cases this can be handled by an XSLT implementation but other instances might include one that returns a binary file based on the information in the record. Transformers might be used in an indexing chain, but are more likely to be used to render a record in a format or schema requested by the end user.

Implementations
  • transformer.XsltTransformer
    Using a configured XSL stylesheet, transform the record into new XML using Fourthought's processor
  • transformer.LibxsltTransformer
    Using a configured stylesheet, transform the record into new XML using libxml2's processor
  • transformer.GRS1Transformer
    Given a record, step through it and turn it into the equivalent GRS1 tree for Z39.50
  • transformer.ChainTransformer
    Given a record, pass it through any number of configured objects to end up with a document.
  • transformer.XmlRecordStoreTransformer
    Transform a record into a string to store in a RecordStore
  • transformer.ClusterExtractionTransformer
    Extract the configured information to cluster and store it locally for later consumption with a documentGroup. Return an empty record.
